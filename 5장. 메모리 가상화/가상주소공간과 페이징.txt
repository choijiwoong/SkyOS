가상주소공간(Virtual Address Space)는 프로세스가 참조할 수 있는 주소의 범위로 프로세스당 하나의 가상주소공간이 주어진다.
윈도우의 경우 프로세스당 응용프로그램 2GB+커널 2GB로 총 4GB의 공간을 제공받는다. 이때 커널의 코드는 여러 프로세스에서 공유한다.

세그멘테이션(GDT)을 통해 얻은 선형주소(4byte)는 Page Directory Table의 인덱스(11bit)와, Page Table Index(10bit)와, offset(11bit)으로 이루어져있다. 
Page Directory Table에서 해당 index에 접근하여 Page Directory Entry(Page Entry Table)를 얻으면, 두번째 해당 인덱스에 접근하여 Page Entry를 얻는다.
해당 Entry의 31~11bit까지 저장된 페이지 기준 주소를 얻고 4KB를 곱한뒤 처음 선형주소에서 offset을 읽어내어 접근하면 실제 물리메모리 주소를 얻을 수 있다.
 프로세스마다 PDT값을 저장하는 PDBR 레지스터 값이 다르고, 다른 프로세스와 메모리 충돌이 발생하지 않도록 적절한 물리 메모리를 매핑한다.
고로 페이징을 활성화하면 프로세스는 특정 물리 메모리 주소에 접근이 제한되고, 특정 프로세스의 메모리 침범으로 인해 다른 프로세스가 망가지는 것을 막을 수 있다.

각각의 테이블은 1024개의 엔트리를 가진다.
각각의 엔트리의 20bit는 4KB단위 정렬된 페이지 테이블 주소 혹은 페이지 기준 주소를 나타내고 나머지는 이용가능성, 페이지 크기, 동시기록, 메모리 존재 여부등을 확인할 수 있는 메타데이터를 갖고있다.
